## ⚠️ ПРАВИЛА — ЗАКОН

**Правила ниже — беспрекословны. Нарушение недопустимо.**

- Помни о них ВСЕГДА, на каждом шаге
- Перед каждым действием — сверяйся с правилами
- Нет исключений, нет "но в этом случае..."
- Нарушил — признай и исправь

---

**Note for Claude agents:**

0. **ГОВОРИ НА РУССКОМ.** Весь диалог ведётся на русском языке.

1. **НЕ ИЗВИНЯЙСЯ.** Просто делай правильно.

2. **НЕ ВЫДУМЫВАЙ.** Не уверен - спроси меня или скажи "не знаю". Не гадай.

3. **CONTEXT7 ДЛЯ ДОКУМЕНТАЦИИ.** Нужна документация библиотеки, фреймворка или API - используй Context7 MCP.

4. **ВОПРОС ОТ ЮЗЕРА ≠ КОМАНДА.**
   - Вопрос — не команда. Отвечай, не делай.
   - Жди подтверждения: "го", "ок", "делай", "да"

5. **ЗАДАЛ ВОПРОС — ЖДИ ОТВЕТА.**
   - Если задал вопрос — БЕЗ ИСКЛЮЧЕНИЙ жди прямого ответа от юзера
   - Никаких косвенных додумываний, только явный ответ

6. **ЧИТАЙ КОД ПОЛНОСТЬЮ.**
   - От первой строки до последней скобки, не останавливайся на "понял идею"
   - Важная логика часто в конце (cleanup, валидация, edge cases)
   - Перед "готово" - проверь что всё покрыто

7. **БОЛЬШИЕ ПРАВКИ - ПО ЧАСТЯМ.** Макс 50-100 строк за раз.

8. **НЕ УПОМИНАЙ AI/CLAUDE** в коммитах, коде, комментариях.

9. **GIT.**
   - Не пуш без команды
   - `--amend` только если не пушил, иначе - спроси разрешения на форс
   - Без прямой команды не делать checkout, revert и прочее меняющее файлы
   - Не `rm` для файлов под гитом - только `git rm` или `git mv`

10. **ПЛАН.**
   - ❌ НИКОГДА не показывай устаревшие планы
   - Задача выполнена? Удали план, не таскай из сессии в сессию
   - Новая задача? Создай новый план с нуля
   - Перед ExitPlanMode — убедись что план про ТЕКУЩУЮ задачу

11. **ARCHITECTURE.md — ИСТОЧНИК ПРАВДЫ.**
   - Перед работой — прочитай `ARCHITECTURE.md`
   - Изменения в архитектуре, новые модули, боты — документируй в `ARCHITECTURE.md` без исключений
   - Не изучай заново то, что уже задокументировано

12. **ПРОВЕРЯЙ АГЕНТОВ.**
   - Агенты могут ошибаться — не передавай их рекомендации слепо
   - Перед предложением фикса — проверь что проблема реальна
   - Сверяй с кодом: может уже есть защита, может проблема надумана

## Code Style

**Naming Conventions:**
- `PascalCase`: Classes, Types, Interfaces
- `camelCase`: Functions, variables, methods
- `UPPER_CASE`: Constants

**Не делать:**
- ❌ Изобретать велосипед - юзай существующие сущности
- ❌ Константы не в `UPPER_CASE`

## Windows-Specific

**Path Separators:**
- Use backslash paths for Claude Code file operations: `H:\bots\Template\src\file.ts`
- Forward slash paths may cause issues

**Bash команды:**
- ✅ Используй forward slashes: `cd /h/bots/Template && git status`
- ❌ НЕ: `cd H:\bots\Template` — бэкслэши = escape
- Для чтения файлов используй Read/Glob/Grep, не cat/find/grep
